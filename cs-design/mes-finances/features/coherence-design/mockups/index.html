DESIGN-004

Standardiser le format des montants (fr-CA)

3 conventions coexistent : $800, $1 500, 1 200,00 $. La norme fr-CA est : espace pour les milliers, virgule pour les decimales, toujours 2 decimales, dollar apres avec espace insecable.

Tableau de conversion — Avant / Apres
Page Avant Apres (fr-CA) Notes
Revenus $800 → 800,00 $ $ avant, pas de decimales
Revenus $4 200 → 4 200,00 $ $ avant
Patrimoine $1 500 → 1 500,00 $ $ avant, pas de decimales
Patrimoine $6 500 → 6 500,00 $ $ avant
Params/charges $1 200,00 → 1 200,00 $ $ avant (decimales OK)
Params/charges $3,99 → 3,99 $ $ avant
Params/revenus 4 200$/mois → 4 200,00 $/mois Pas d'espace, pas de decimales
Hero headers $847 → 847,00 $ Meme convention partout
Cas extreme $0 → 0,00 $ Jamais vide, jamais $0
Cas extreme -$125 → -125,00 $ Signe devant le nombre
Cas extreme $125000 → 125 000,00 $ Espaces milliers multiples
Implementation : Helper centralise formatCAD(amount) base sur Intl.NumberFormat('fr-CA', { style: 'currency', currency: 'CAD' }). Utiliser \u00A0 (espace insecable) entre le montant et le $ pour eviter les retours a la ligne. Toujours 2 decimales, meme pour les montants ronds. Hero headers : meme format, la taille de police fait le monument, pas le format.
